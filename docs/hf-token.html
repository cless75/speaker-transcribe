Получение токена Hugging Face

Токен нужен для диаризации (распознавания спикеров) — модели pyannote закрыты (gated) и требуют авторизации. Для чистого ASR без диаризации (--speaker-mode off) токен не нужен.

1Аккаунт

Зарегистрируйтесь или войдите на huggingface.co.

2Принять условия gated-моделей

Под тем же аккаунтом откройте обе страницы и нажмите «Agree and access repository»:

  • pyannote/segmentation-3.0
  • pyannote/speaker-diarization-3.1
Без принятия условий токен не даст доступа к моделям — диаризация упадёт с ошибкой 403.

3Создать токен

  1. Settings → Access Tokens → New token.
  2. Тип — Read. Имя — любое (напр. speaker-transcribe).
  3. Скопируйте значение hf_... (показывается один раз).

4Задать токен на машине

Движок читает токен из переменной окружения HF_TOKEN — это надёжнее, чем CLI-логин (не зависит от наличия huggingface-cli, переживает пересоздание venv).

Windows · на текущую сессию PowerShell (сразу работает):

$env:HF_TOKEN = "hf_ВАШ_ТОКЕН"

Windows · постоянно (для новых окон):

setx HF_TOKEN "hf_ВАШ_ТОКЕН"
# затем откройте НОВОЕ окно терминала — переменная появится только в новых процессах

macOS / Linux:

export HF_TOKEN="hf_ВАШ_ТОКЕН"
# чтобы сохранить — добавьте строку в ~/.zshrc или ~/.bashrc
Альтернатива — CLI-логин (пишет токен в ~/.cache/huggingface/token): ...\venvs\asr\Scripts\python -m huggingface_hub.commands.huggingface_cli login. Код проверяет env-переменную в первую очередь, поэтому способ выше проще.

5Проверка

После установки токена запустите транскрибацию с диаризацией (--speaker-mode on) — она скачает модели pyannote и не упадёт на 403.

Безопасность: токен — это секрет. Никогда не коммитьте его в git и не кладите в публичные папки/облако. Храните в переменной окружения или приватном менеджере секретов.
